IELTS Hub — Development Roadmap

From Tracker to the Ultimate Language Learning Hub
v8 · 2026-04-09 · 30/30 Shipped · 30 Ideas
1

Product Philosophy

This is a Hub — Not a Classroom

IELTS Hub is where learners track, organize, analyze, and own their study data. We don't teach — we give learners the best workspace to teach themselves. Think Notion for language learning: editable, interactable, data-driven.

✎

Editable Everything

Every piece of data the user enters should be inline-editable. No modal hell, no page redirects. Click, type, done.

📊

Data Ownership

Users own their data. Export, visualize, slice it any way. The hub exists to surface insights they can't get elsewhere.

⚙

Interactable Workspace

Not static pages — interactive widgets, toggles, drag-and-drop. Every screen is a tool, not a report.

🎯

Niche-Deep, Not Broad

Go deeper into IELTS/language learning than any generic notes app can. Question types, band calculations, spaced repetition — all baked in.

Phase A — Sharpen the Blade

Goal: Polish existing features to be best-in-class before adding new ones.

These ideas enhance what's already built — deeper analytics, smoother UX, faster navigation. Low risk, high impact. Ship first.

P0 Analytics ✓ Shipped Deep Analytics Dashboard

Transform the current analytics page from a summary view into an interactive data exploration tool. Users can filter by date range, module, question type, and session — then see trend lines, distribution histograms, and comparative charts that update in real time.

Impact
Effort
Why: The existing heatmap + basic charts leave too much data on the table. Power users (the hub's core audience) want to slice their data freely. This is what separates a hub from a tracker — Notion-level depth applied to IELTS analytics.

P0 UX Inline Editing Everywhere

Make every data point — scores, notes, dates, question types — clickable and editable directly in tables, cards, and lists. No more "open detail page, find field, edit, save, go back." Click a cell, type, press Enter. Undo supported. This applies to session logs, vocab entries, grammar notes, and all tracking tables.

Impact
Effort
Why: "Editable everything" is the number one differentiator from static test-prep sites. This is what makes it feel like Notion — a workspace you control, not a read-only dashboard. Every extra click to edit is a user lost.

P0 Analytics ✓ Shipped Readiness Score (True Band Estimate)

A calculated metric that estimates the user's "real" band by factoring in un-attempted question types (weighted by official exam frequency), consistency, recent trend direction, and data sufficiency. Displayed as a secondary metric below the user's tracked band average. Types with fewer than 3 data points show as "insufficient data" rather than zero.

Impact
Effort
Why: Most learners only practice question types they're comfortable with. The Readiness Score exposes blind spots honestly — but uses official frequency weighting and a "Readiness" framing to motivate rather than demoralize. Named "Readiness" instead of "True Average Band" to avoid crushing confidence.

P1 UX ✓ Shipped Global Search / Cmd+K

A spotlight-style search palette (Cmd+K / Ctrl+K) that searches across all data: vocab entries, grammar notes, session logs, topics, settings. Results grouped by type with keyboard navigation. Also serves as a quick-action launcher: "Add vocab," "Log listening session," "Go to analytics."

Impact
Effort
Why: As the hub grows, finding things becomes the bottleneck. Notion, Linear, Raycast — every serious productivity tool has Cmd+K. This is table stakes for a hub that wants to replace scattered tools.

P1 Analytics ✓ Shipped Confidence Trend Lines

Per-question-type trend lines showing accuracy over time with a smoothed moving average. Visible on both the analytics page and individual module pages. Color-coded: improving (green slope), stagnating (yellow flat), declining (red slope). Helps users see if their practice is actually working.

Impact
Effort
Why: Current heatmaps show a snapshot — users can't tell if they're getting better or worse at a specific type. Trend lines close this gap and give emotional reward ("I'm improving!") or early warning ("This is slipping"). Recharts makes this low-effort.

Phase B — Expand the Hub

Goal: Add new hub capabilities that make IELTS Hub the single destination for all study activities.

These are new features that turn the app from a tracker into a workspace. Each one adds a "reason to come back" and reduces the need for external tools.

P0 Hub ✓ Shipped Resource Library

A DB-backed, taggable collection where users save links to practice tests, YouTube channels, podcasts, articles, and study materials — organized by module, difficulty, and custom tags. Each resource shows a usage counter, personal rating, and notes. Bulk-add via paste. Admin can seed curated starter resources.

Impact
Effort
Why: Every IELTS learner maintains scattered bookmarks across browsers, notes apps, and chat groups. A built-in resource library with IELTS-specific tagging (module, question type, band level) is something no generic bookmark manager offers. This is a core hub feature — centralizing everything in one place.

P1 Hub ✓ Shipped Calendar Planner

A weekly study planner where users block out study sessions, assign modules/topics, and track completion. MVP: weekly grid view with checkboxes per slot. Phase 2: month view. Phase 3: AI-suggested study plan based on weaknesses and exam date. Integrates with the streak system.

Impact
Effort
Why: "What should I study today?" is the #1 question learners ask. A visual planner answers this daily and creates a habit loop (plan → execute → check off → see streak). Scoped to weekly grid MVP to avoid scope creep — full calendar can come later.

P1 Content ✓ Shipped Exam Quarter Awareness

Ambient awareness of the current IELTS exam cycle: which speaking topics are "in season" (current quarter forecast), countdown to the user's exam date, and study urgency indicators. The forecast data already exists in the DB (seed-forecast-q2-2026.sql) — this surfaces it prominently in the dashboard and speaking module.

Impact
Effort
Why: The forecast seed data is already in the DB but barely surfaced. Making it prominent gives users exam-relevant context without them having to research elsewhere. Low effort, high relevance — this is exactly what a hub should do: aggregate context.

P2 Content ✓ Shipped Question Type Wiki

A built-in reference for each of the 41 tracked question types: what it looks like, common traps, strategies, official frequency, and links to the user's own performance data for that type. Editable — users can add their own notes per type. Serves as a quick-reference during practice.

Impact
Effort
Why: The app tracks 41 question types but doesn't explain what they are. New users (and even experienced ones) need a quick reference. Making it editable turns it from a static FAQ into a personal strategy notebook per type — hub behavior, not tutorial behavior.

P2 Hub ✓ Shipped Cross-Module Linking

Connect data across modules: link a vocab entry to a writing essay where it was used, link a grammar rule to listening errors it caused, link a speaking topic to relevant reading passages. Bi-directional — view related items from either side. Think wiki-style [[backlinks]].

Impact
Effort
Why: IELTS skills are interconnected — a weak grammar point affects writing, listening, and speaking. But current tools treat each module as a silo. Cross-linking is what makes a hub a hub, not just four separate trackers glued together. This is the Notion/Obsidian-style linking applied to language learning.

P2 Analytics ✓ Shipped Weekly Study Digest

An auto-generated weekly summary: total study time, sessions completed, band movement, streak status, top-improved type, weakest type, and one suggested focus for next week. Displayed as a dismissible card on the dashboard every Monday. Optionally emailable.

Impact
Effort
Why: Weekly digests are the "glue" that keeps users coming back. Without them, users must manually assess progress. A pre-built weekly summary turns passive data into an active nudge — and the suggestion engine already has the data to power this.

Phase C — Power Tools

Goal: Add differentiating features that make IELTS Hub irreplaceable for serious learners.

These are heavier features that require more effort but create strong moats. Each one gives users a reason to stay that's hard to replicate elsewhere.

P1 Hub ✓ Shipped Mistake Journal

A dedicated section to log, categorize, and review mistakes across all modules. Each entry captures: what the mistake was, why it happened (carelessness, knowledge gap, time pressure), the correct approach, and which question type it occurred in. Supports tagging, filtering, and spaced-repetition-style review of past mistakes.

Impact
Effort
Why: "Error journals" are recommended by every IELTS tutor but no digital tool supports them well. Learners use paper or random notes apps. A structured mistake journal with tagging and spaced review is a killer feature — it turns mistakes into a study system, not just regrets.

P2 UX ✓ Shipped Study Timer / Pomodoro

Built-in study timer with Pomodoro mode (25/5 or custom intervals). Auto-logs study time to the relevant module. Persists across page navigation. Shows in a floating mini-widget or status bar. Integrates with daily activity tracking and streak calculations.

Impact
Effort
Why: Learners currently switch to separate timer apps (Forest, Toggl, phone timers). An integrated timer that auto-logs to the right module eliminates that context switch. It also makes study time tracking automatic instead of manual — better data for analytics.

P2 Analytics ✓ Shipped Writing & Speaking Heatmaps

Extend the existing heatmap visualization (currently Listening + Reading only) to cover Writing and Speaking. Writing: heatmap by task type (Task 1 types: line graph, bar chart, pie, table, map, process, mixed; Task 2: opinion, discussion, problem-solution, two-part, advantage-disadvantage). Speaking: heatmap by Part 1/2/3 topic categories.

Impact
Effort
Why: The heatmap is one of the most visually powerful features but only covers half the exam. Extending it to Writing and Speaking gives users the same "weakness at a glance" insight for all four modules. The visualization code already exists — this is mostly data modeling.

P2 Core ✓ Shipped Batch Import

Import data from CSV/spreadsheets: past test scores, vocab lists, grammar notes. Supports mapping columns to fields. Validates data before import. Essential for users migrating from spreadsheets or other tools. Also enables bulk-add of vocab (paste a list of words).

Impact
Effort
Why: The biggest friction for a new hub is "I already have data elsewhere." Batch import removes that barrier. Users who already track IELTS scores in Excel can switch without losing history. This is also competitive — most IELTS apps have no import feature at all.

P2 Social ✓ Shipped Tutor / Shared View

Generate a read-only shareable link to the user's analytics and progress. Tutors can see the student's heatmap, trend lines, mistake patterns, and practice frequency without needing an account. Time-limited link (expires after 7 days by default). No editing access.

Impact
Effort
Why: Many IELTS learners work with tutors (online or offline). Currently they screenshot their data. A shareable read-only view is a viral growth mechanism — every tutor who sees it becomes aware of the product. It also positions the hub as a professional tool, not just a student toy.

P2 AI ✓ Shipped Speaking Recorder & Self-Review

Record speaking practice directly in the hub, then replay and self-annotate: mark hesitations, fillers, pronunciation issues, and good moments. AI provides optional fluency metrics (words per minute, filler count, pause length). Saved recordings linked to topic and date. Framing: "Record and analyze YOUR speaking" — a journaling tool, not a test simulator.

Impact
Effort
Why: Speaking is the hardest module to self-study because learners can't hear themselves objectively. Recording + self-annotation is a proven journaling technique. AI metrics add objective data. Careful framing as a "recorder" (hub behavior) not a "test" (teaching behavior) keeps it on brand.

P2 Core ✓ Shipped Export to PDF

Generate a polished PDF report of the user's progress: overall bands, module breakdowns, heatmaps, trend charts, streak data, and key recommendations. Useful for visa applications (proof of IELTS preparation), tutor meetings, and personal records. Customizable: choose which sections to include.

Impact
Effort
Why: "Data ownership" means users can take their data out. A well-designed PDF export is practical (tutor meetings, visa docs) and psychologically satisfying (tangible proof of progress). It also reinforces trust — the hub isn't locking data in.

Phase D — Ecosystem Play

Goal: Long-term features that expand the hub into a full ecosystem — social, customizable, multi-platform.

These ideas shape the product's future identity. Each is individually valuable but best built on top of the solid foundation from Phases A-C. Prioritize based on user demand signals.

P2 Hub ✓ Shipped Goal-Setting Engine

Users set target bands per module and a target exam date. The system calculates required improvement rate, shows a progress-to-goal gauge, and warns if the user is falling behind pace. Goals are visible on the dashboard and influence the recommendation engine's urgency factor.

Why: Goals give structure to practice. Without them, users drift. A visual "you need to improve 0.5 bands in 6 weeks" creates urgency and focus — exactly what the hub should provide as a planning tool.

P2 Hub ✓ Shipped Rich-Text Notes (Mini Notion)

Upgrade text fields across the app from plain text to a lightweight rich-text editor: bold, italic, bullet lists, headings, code blocks, and inline images. Most impactful in vocab definitions, grammar notes, essay drafts, and the mistake journal. Not a full Notion clone — just enough formatting to make notes useful.

Why: Plain text notes feel limiting — users want to bold key points, add example sentences with formatting, and structure their thoughts. Rich text is the bridge between "tracker" and "workspace." Scope carefully: Tiptap or Slate.js as editor, not a custom solution.

P3 UX ✓ Shipped Smart Notifications

In-app and optional push notifications: spaced repetition reminders, streak danger alerts, weekly digest nudges, exam date countdowns, and "you haven't practiced [module] in X days" prompts. Configurable — users choose which notifications to receive and when.

Why: Retention depends on re-engagement. Thoughtful, configurable notifications bring users back at the right moment. The key is "smart" — not spammy. Notifications tied to actual user behavior (declining streak, overdue vocab review) feel helpful, not annoying.

P3 Social ✓ Shipped Achievement Sharing

Generate shareable cards (images) for milestones: new band achieved, streak record, badge earned, level up. Designed for Instagram Stories, Twitter, and study group chats. Branded with IELTS Hub watermark. One-click share from the achievement modal.

Why: Gamification badges are already built but have no external reach. Shareable cards turn every achievement into organic marketing. IELTS study groups on Facebook and Telegram are massive — every shared card is a potential new user.

P2 UX ✓ Shipped Dashboard Toggles / Layout Control

Let users show/hide dashboard widgets and rearrange their order. Some users want dense data; others want a clean overview. Persist layout preferences per user. MVP: toggle visibility of each widget card. Future: drag-and-drop reordering.

Why: "Interactable workspace" means users control their view. A one-size-fits-all dashboard frustrates both data-heavy users and minimalists. This is a small feature with outsized "it feels like MY tool" impact.

P3 Hub ✓ Shipped Multi-Source Aggregation View

A unified "recent activity" feed that combines data from all modules: last 3 listening sessions + 2 vocab entries + 1 writing draft + today's grammar review. Single scrollable timeline. Filter by module or date. Think GitHub activity feed but for IELTS study.

Why: Currently each module is its own silo. A unified activity view shows the "full picture" of study behavior across all four skills — something no module page can do alone. This is the hub's home screen story.

P3 UX ✓ Shipped Focus Mode

A distraction-free mode that hides the sidebar, nav, and all non-essential UI — leaving only the current task (writing an essay, reviewing vocab, logging scores). Activated via keyboard shortcut or toggle. Pairs well with the study timer.

Why: Power users don't want chrome when they're in flow. Focus mode is a small UX touch that signals "this is a serious workspace tool" — the same reason Notion, iA Writer, and VS Code all have distraction-free modes.

P3 UX ✓ Shipped Customizable Dashboard Widgets

Beyond toggles — let users configure what each dashboard card shows. E.g., the "Listening" card could show either "recent sessions" or "weakest question types" or "accuracy trend." Widget library with preset options per card. Drag-and-drop grid layout.

Why: The ultimate expression of "interactable workspace." Extends Dashboard Toggles into full customization. High effort, but transforms the dashboard from a page into a personal control panel. Build after validating that Dashboard Toggles are actually used.

P3 Hub ✓ Shipped Multi-Exam Support (TOEFL, PTE, Duolingo)

Abstract the question type system to support other English proficiency exams. Same hub, same UX — different question type taxonomies and scoring systems. Users choose their exam on signup. Data stays unified for cross-exam comparison. Start with TOEFL iBT (closest format), then PTE Academic.

Why: Massively expands TAM (Total Addressable Market) without rebuilding the product. Many learners take both IELTS and TOEFL. The hub architecture (question types, band calculations, heatmaps) is already abstract enough — most work is just new lookup tables. But only do this after the IELTS experience is rock-solid.

P3 Core ✓ Shipped Offline-First PWA

Service worker with IndexedDB cache for offline access to vocab review, grammar notes, and score logging. Syncs when back online. PWA manifest for "Add to Home Screen" on mobile. Essential for learners in areas with unreliable internet or who study during commutes.

Why: IELTS learners are global — many study in locations with spotty internet. Offline-first is also a competitive moat: most web-based IELTS tools are fully online. This is technically complex but unlocks mobile usage without building native apps.

P3 Social ✓ Shipped Study Groups

Create or join small study groups (4-8 people) with shared progress dashboards, group streaks, and a simple group chat. Members see each other's weekly digests (opt-in). Group leaderboards for friendly competition. Built on existing Supabase real-time capabilities.

Why: IELTS study is often social — Facebook groups, Telegram chats, study buddy arrangements. Bringing this into the hub creates network effects and retention. Groups keep users accountable. But this is a Phase D feature because it needs the core experience to be polished first.

P3 Social ✓ Shipped Peer Review Exchange

Users submit Writing Task 2 essays for peer review and review others' essays in return. Structured feedback form (not free text) based on IELTS criteria: Task Response, Coherence, Lexical Resource, Grammar. Earn XP for reviewing. Moderated with quality gates.

Why: AI essay grading exists but human feedback is different. Peer review creates community engagement, gives users practice at evaluating writing (which improves their own), and generates content for the platform. But heavy moderation needed — save for when the user base justifies it.

Reading this Matrix

Priority tiers: P0 = build immediately (next sprint), P1 = build soon (this month), P2 = build when stable (next month), P3 = build when validated (backlog).

Impact = how much this moves the product toward "ultimate language learning hub." Effort = implementation complexity (more stars = more work).

Best bang for buck: Readiness Score (P0, high impact, low effort), Exam Quarter Awareness (P1, mid impact, very low effort), Confidence Trend Lines (P1, mid impact, low effort). These three can ship in a single sprint and collectively make the analytics story dramatically stronger.
